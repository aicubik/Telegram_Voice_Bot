## 🔗 OpenRouter Consultation Protocol

Система автоматически использует внешние модели через `mcp_openrouter_chat_completion`, когда внутренние ресурсы недостаточны.

### Матрица внешних моделей (Verified Free)

| Роль | ID Модели | Когда использовать |
| :--- | :--- | :--- |
| **Expert Analyst** | `meta-llama/llama-3.3-70b-instruct:free` | Сложная бизнес-логика, Kotlin/Java архитектура. |
| **Data Context** | `stepfun/step-3.5-flash:free` | Чтение и саммаризация больших файлов (>200К токенов). |
| **Logic Tester** | `liquid/lfm-2.5-1.2b-thinking:free` | Поиск сложных багов, режим рассуждений (thinking). |
| **UI Critic** | `google/gemma-4-31b-it:free` | Анализ дизайна, скриншотов и верстки. |
| **Default Router** | `openrouter/free` | Все прочие задачи или когда основная модель недоступна. |

### Триггеры автоматического вызова
1.  **Сложность:** Задача требует знаний, выходящих за рамки GPT-4o-mini (например, редкие либы Android).
2.  **Self-Check Fail:** Если скрипт или правка не сработали дважды — запрашиваем "второе мнение".
3.  **Архитектура:** Перед созданием новых файлов и рефакторингом основной кодовой базы.

## 🔍 Модуль Self-Check + OpenRouter

При обнаружении ошибки:
├─→ Попытка 1: Анализ ошибки → автоисправление.
├─→ Попытка 2: Пересмотр подхода → автоисправление.
**├─→ Попытка 3: Консультация с `meta-llama/llama-3.3-70b-instruct:free`.**
│   └─→ Промпт: "Я — агент, у меня ошибка [X] в файле [Y]. Проанализируй и дай точное решение."
└─→ СТОП → Отчет пользователю.

## 🔐 Безопасность и Лимиты
- НИКОГДА не передавать токены/ключи в промпты OpenRouter.
- Использовать только `:free` версии моделей для экономии бюджета пользователя.
- Если модель возвращает 429 (Too Many Requests) — ждать 30 секунд или переключиться на другую модель.
